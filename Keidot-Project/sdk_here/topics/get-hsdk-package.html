Get the HERE SDK package

Get the HERE SDK package

To start developing with the HERE SDK, you must get the HERE SDK package, register an app, and create app credentials to authenticate your application.

The HERE SDK package contains:

  • The HERE SDK plugin folder which contains the HERE SDK binary.
  • Several documentation assets and example apps.
  • The HERE_NOTICE file that must be included in any app created with the HERE SDK. See the open source notices to learn more.

Follow these steps to get started:

  1. Sign in or create a free account on the HERE platform portal. Note that this requires a valid credit card. To learn more about the HERE platform, read this blog post.
  2. Download the HERE SDK package for your edition from this page.
  3. Register an app and get your app credentials (Access key ID and Access key secret) for use with the Explore Edition.
    • Sign in to your account on the HERE platform portal.
    • Open the Launcher from the top navigation bar and go to Access Manager.
    • In the Apps view, click Register new app.
    • Provide the app name, the optional app description, and click Register.
    • On the Credentials tab, select OAuth 2.0 and select Create credentials.
    • Save the credentials for later use. You can't get the Access key secret after you close the window. Click Download to save the credentials to a file on your machine.
  4. Choose the right pricing plan for your needs. By default, the Explore Edition can be used with the HERE Base Plan. To learn more, read the FAQs. Contact us if you want to switch to a different plan.

Note

Depending on the selected plan, you're charged for using the HERE SDK based on the number of transactions per month and/or other parameters such as monthly active users (MAU).

About your credentials

Obtain your credentials as shown above. The HERE SDK requires two strings to authenticate your app:

  • Access key ID (access_key_id): A unique ID for your account.
  • Access key secret (access_key_secret): A secret key.

No other credentials or tokens are needed to use the HERE SDK.

Find tips in the Best Practices section on how to protect your credentials.

Your credentials will work with all example apps for your edition you can find on GitHub. The credentials for the Navigate Edition will work for all apps, but credentials for the Explore Edition will not work for all features of the Navigate Edition.

When you obtain your credentials, also a here.client.id or an app ID is shown. These IDs are not needed by the public APIs of the HERE SDK, but they can be useful when contacting the HERE support team.

See also the below sections to learn more details on how to set the credentials for an app. More information on managing your apps can be found in the Identity & Access Management guide.

App credentials and project scope

When you plan to use the same credentials for multiple apps, then you also need to set a scope to differentiate your apps in our system. If you use your credentials only for a single app, then this is not needed.
