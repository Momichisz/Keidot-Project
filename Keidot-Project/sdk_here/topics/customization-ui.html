User interface components

User interface components

Enhance your application's user interface with the HERE SDK, from utilizing the HERE Icon Library to building essential UI components and understanding the differences between visual features and headless usage.

How to use the HERE Icon Library

You can use the icons from the open-source HERE Icon Library, based on the provided license, in your own commercial and non-commercial applications.

You can clone the repository with:

git clone https://github.com/heremaps/here-icons.git

Alternatively, you can directly download the individual assets from GitHub, as per your needs. Note that the library is just a collection of images, so no app-integration is offered.

Most icons are available in different formats such as PNG and SVG, and in various resolutions.

Additionally, the icons come in both solid and outline formats.

UI building blocks

With a few exceptions, such as the VisualNavigator which is available only for the Navigate Edition, the HERE SDK does not provide UI or UI assets for building your own apps. Most of the information available is given as simple data types.

  • However, reusable assets for use in your own applications can be found in the MSDKUI open source project from HERE - available on GitHub under this link. The MSDKUI project has been deprecated as it is only compatible with the deprecated 3.x version of the HERE SDK, but it can be still useful to look into the available UI components as they have been designed to be decoupled from any SDK logic.

  • Alternatively, the reference application for the HERE SDK for Flutter (Navigate Edition) shows many UI assets in a demo application.

Both projects above are sourced from the HERE Icon Library.

At the moment, we provide no further guidance on how to integrate the available building blocks - stay tuned.

Visual features vs headless usage

The modular structure of the HERE SDK allows for flexibility in how you utilize its components, such as search or routing, independently of displaying a map view. This means you can integrate various functionalities into your app without showing a map or even use a map view from a different vendor. For instance, you can build a headless application that leverages the HERE SDK's search and routing capabilities or combine its routing features with maps from other providers.

Visual features of the HERE SDK include:

  • Visual Navigator: The VisualNavigator provides real-time, on-screen navigation guidance, including turn-by-turn directions and visual cues to assist users in reaching their destinations.
  • IconProvider: This feature generates road shields and other icons, ensuring that map elements are visually consistent and informative.
  • Realistic views: With the RealisticViewWarningListener you can receive SVG string data for signpost shields and complex junction views in 3D. The RealisticViewWarning event contains SVG data for both, signposts and junction views.
  • HERE Style Editor: Customize the appearance of your maps with the HERE Style Editor, which allows you to adjust colors, icons, and other visual elements to match your application's branding and user experience.

Note that not all features are available for all editions. However, the modular nature of the HERE SDK ensures that you can build a full visual experience or integrate the available backend services in a headless environment.
